discretionary access control (DAC)

A type of access control where the owner of a resource determines who can access it.

Description

Discretionary Access Control (DAC) is a security model that allows resource owners to determine who can access their resources. This model provides flexibility by enabling users to grant or deny access to their resources, such as files or directories, based on their discretion. In DAC, permissions are assigned to users or groups, allowing them to control access to their own files. This means that if a user has the right to a resource, they can share it with others at their discretion. While this flexibility can enhance usability and collaboration, it can also lead to security vulnerabilities, as users may inadvertently grant access to unauthorized individuals. DAC is commonly implemented in operating systems and applications, where file permissions are set by users. However, this model can be less secure than mandatory access control (MAC) systems, where access permissions are enforced by a central authority, as DAC relies heavily on the user's choices and behavior.

Examples

  • A user shares a folder on a network drive with specific colleagues.
  • A file owner sets permissions to allow others to read or modify their document.

Additional Information

  • DAC is commonly used in operating systems like Windows and UNIX/Linux.
  • It can be combined with other access control models, such as Role-Based Access Control (RBAC), for enhanced security.
