distinguished name (DN)

A unique identifier for an entry in a directory service, composed of a sequence of attribute-value pairs.

Description

In the context of network security and directory services, a distinguished name (DN) is a string that uniquely identifies an entry in a directory information tree (DIT). A DN is composed of multiple components, each representing an attribute of the entry, such as the common name (CN), organizational unit (OU), organization (O), and country (C). The components are structured in a hierarchical manner, with the most specific identifier listed first and the least specific last, typically separated by commas. For example, a DN for a user in a corporate directory might look like 'CN=John Doe, OU=Sales, O=ExampleCorp, C=US'. DNs are crucial in various applications, including LDAP (Lightweight Directory Access Protocol), where they are used to locate and manage resources within the directory. They enable precise identification of entries, which is essential for access control and authentication mechanisms in network security.

Examples

  • CN=Jane Smith, OU=Engineering, O=TechCorp, C=US
  • CN=Alice Brown, OU=Marketing, O=GlobalInc, C=UK

Additional Information

  • DNs are case-insensitive but should be used consistently.
  • They play a vital role in access control and authentication in directory services.
