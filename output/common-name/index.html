common name

The common name (CN) is an attribute in a digital certificate that identifies the entity associated with the certificate.

Description

In the context of network security, the common name (CN) is a crucial component of digital certificates, which are used in various protocols such as SSL/TLS for establishing secure communications over the internet. The common name typically represents the fully qualified domain name (FQDN) of the server that the certificate is issued for. For instance, if a website's address is 'www.example.com', the common name in its SSL certificate would be 'www.example.com'. If the certificate is meant to cover multiple subdomains, a wildcard character may be used (e.g., '*.example.com'). The common name helps clients verify that they are communicating with the correct server and not a malicious entity. It plays a significant role in the authentication process, ensuring that the entities involved in a communication can trust each other's identities. Misconfigurations or mismatches in the common name can lead to security warnings and potential vulnerabilities, emphasizing the importance of correctly setting and validating this attribute in digital certificates.

Examples

  • CN: www.example.com for a website's SSL certificate.
  • CN: *.example.com for a wildcard SSL certificate covering all subdomains.

Additional Information

  • The common name must match the domain name being accessed to avoid security warnings.
  • In addition to the common name, certificates may include Subject Alternative Names (SAN) for additional domain validation.
