evaluated system

A system that has undergone assessment to determine its security capabilities.

Description

In the context of network security fundamentals, an evaluated system refers to a computing system that has been rigorously assessed against a defined set of security standards and criteria. This evaluation typically involves testing the system's security features, policies, and controls to ensure they meet specified requirements for confidentiality, integrity, and availability. Evaluated systems are often subjected to formal evaluation processes, such as those outlined in the Common Criteria for Information Technology Security Evaluation (CC), which provides a framework for evaluating the security properties of IT products. The result of this evaluation is usually a certification or assurance level that indicates the degree of trust that can be placed in the system's security. Organizations often prefer evaluated systems for critical applications, as they provide a higher degree of confidence in protecting sensitive information from unauthorized access and attacks.

Examples

  • A firewall that has been evaluated and certified against EAL4 standards.
  • An operating system that has undergone Common Criteria evaluation for secure deployment in government environments.

Additional Information

  • Evaluated systems are often required in government and military applications.
  • The evaluation process can be costly and time-consuming, but it enhances security assurance.
